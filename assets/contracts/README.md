# 契約書PDF配置手順

## 概要
KACP各プランの契約書PDFをこのフォルダーに配置してください。

## 必要なファイル
以下の3つのPDFファイルを配置してください:

1. `contract-home.pdf` - KACP HOME（家庭向け）プラン用
2. `contract-business.pdf` - KACP（事業者向け）プラン用
3. `contract-one.pdf` - KACP ONE（単発）プラン用

## ファイル形式
- 形式: PDF
- 推奨サイズ: 5MB以下（読み込み速度のため）
- ページ数: 制限なし（通常2〜10ページ推奨）

## 配置方法
1. 用意した契約書PDFを上記のファイル名にリネーム
2. このフォルダー（`c:\LP\assets\contracts\`）にコピー
3. ページをリロードして動作確認

## 動作確認手順
1. ブラウザで `c:\LP\index.html` を開く
2. 料金プランセクションで任意のプランの「このプランを選ぶ」ボタンをクリック
3. モーダルが開き、PDFが表示されることを確認
4. 署名フォームに入力し、「署名して申し込む」ボタンをクリック
5. 確認メッセージが表示されることを確認

## トラブルシューティング

### PDFが表示されない場合
- ファイル名が正しいか確認（スペルミス、大文字小文字）
- PDFファイルが破損していないか確認
- ブラウザのコンソール（F12 → Console）でエラーを確認
- ローカルサーバー経由で開く（ファイルURLの制限回避）:
  ```powershell
  python -m http.server 5173 -d C:\LP
  ```
  → http://localhost:5173/index.html で開く

### モーダルが開かない場合
- ブラウザのコンソールでJavaScriptエラーを確認
- `script.js` が正しく読み込まれているか確認

## セキュリティに関する注意
- 契約書PDFには個人情報や機密情報を含めないでください
- このフォルダーは公開Webサイトでは適切なアクセス制限をかけてください
- 本番環境では署名データをサーバー側で安全に処理・保存してください

## 署名データの処理（カスタマイズ）
現在の実装では署名データはブラウザ上でアラート表示のみです。
本番運用時は `script.js` の以下の箇所を編集してください:

```javascript
// script.js 内の署名フォーム送信処理
// 例: await fetch('/api/contract-signature', { method: 'POST', body: JSON.stringify(formData) });
```

サーバー側でデータを受け取り、以下を実装することを推奨:
- データベースへの保存
- 確認メールの自動送信
- PDFへの電子署名埋め込み
- 管理者への通知

## 参考: サンプル契約書テンプレート
実際の契約書は法務担当者と相談の上作成してください。

必須記載事項の例:
- サービス名称とプラン内容
- 契約期間と更新条件
- 料金と支払い方法
- サービス提供範囲
- 免責事項
- 解約条件
- 個人情報の取り扱い
- 契約者と事業者の情報
